Before making HTML layouts, you need to understand the normal document flow of HTML pages, and know how to use the following CSS properties: float, position, and display.

In this article, we'll look at just what this normal document flow is, and how the CSS properties of display and float work with it.

Normal document flow

We can think of the normal document flow as a river of content (i.e. a "river" of HTML elements). All the content on our page flows down this "river", left-to-right, then top-to-bottom.

It is this default "flow" that determines how an HTML element will be displayed on a page.

The caveat here is that some content will take up the full width of the screen, i.e. the full "left-to-right" portion of the "river".

Many HTML elements behave this way, and the div element is one of them. These elements are referred to as block level elements. Regardless of how wide a block element's content actually is, it will always take up the full width of the screen, and whatever follows it, will stack under it.

Another way to think of it is that block level elements have an implicit line break at their edge.

That's why the div element is referred to as a block-level element. When we say that an HTML element is a block-level element, we are referring to it's display property.

In other words, some elements, referred to as block-level elements, will take up the full width of the available screen space, not allowing any other element to fit on either their left side, or their right side.

This leads us to an obvious question: Given the default behavior of block-level elements, including divs, how do we then fit two divs next to each other?

Ways to place two divs side-by-side

There are many ways to achieve this. However, since this article is dealing with the display, float, and normal document flow, we will focus on these.

Specifically, we will look at two different ways of making 2 divs fit side-by-side, rather than stack on top of one another. These two ways are:

  • Changing the value of the display property
  • Using floats

Changing the value of the display property to position two divs next to each other

Before we dive into this solution, it is important to note that there are many possible values that we can give our CSS display property. While divs and some other elements have the display: block set by default, some other HTML elements have display: inline set as their default.

Thus, by virtue of changing the display value to something other than "block", we can get our desired effect, as can be seen in the next example:

In the example above, we have four divs. On the first two divs, we assigned display:inline, which places our divs next to one another.

The above example also shows us the interesting behavior of margins, paddings, and dimensions, on elements that have display property set to inline.

What happens with margins on DIVs that have display: inline?

Besides not spanning the full width of the screen, there are also other differences between block level elements and inline elements. What follows is a list of things that are specific to inline elements:

  • Setting the width and height property on inline elements will not change anything.
  • Setting the margin property on inline elements will only affect the horizontal axis.
  • Setting the padding property on inline elements will affect both the x and the y axis, but will not work as expected because display: inline warps the box model.

Let's see an example of setting these properties on inline elements:

See the Pen Using display: inline on divs AND height, width, padding, and margin by Ajdin (@AjdinImsirovic) on CodePen.

As we can see above, the width and height are not honored. The margins are honored only for the horizontal axis, and the padding sort of works in all directions.

However, since the box model on inline elements is essentially broken (or at least, significantly warped), it is best to assign padding, margin, and dimensions only to inline blocks' wrapping elements, and thus avoid these issues.

Introducing the inline-block value for the display property

So far, we've seen the display: block and the display: inline property. However, there are quite a few more.

To become familiar with what is available, let's list all the possible values of the display property:

  • box, inline-box, block, contents, flex, flow-root, grid, inherit, initial, inline, inline-block, inline-flex, inline-grid, inline-table, list-item, none, table, table-caption, table-cell, table-column, table-column-group, table-footer-group, table-header-group, table-row, table-row-group, unset

In the above list, there are 26 possible values for the display property.

At this point, we will focus on inline-block.

The benefit of using inline-block is simple: it supports using the box model. However, it also has downsides, namely in obeying whitespace.

Put differently, using inline-block on divs allows you to use a div as an inline element that honors the box model, but that element will also honor whitespace.

Here is what we get in practice:

The result is just what we need: the inline-block elements are not honoring whitespace, which is the behavior that we'd expect in the first place.

Making sense of 26 possible display property values

At this point, we have looked at display: block, display: inline, and display: inline-block.

How do we make sense of so many other possible values? It's simple, we'll divide and conquer!

Instead of looking at the long list of values, we will group them in the following way:

Looking at the table above, it is much easier to reason about the display property now.

Basically, what we have done is, we have divided all the values into logical, meaningful sub-groups:

  • Block: includes the "regular" block, inline-block, and inline values
  • Flex and Grid: inludes the more recent CSS layout specifications
  • Table: all the values we can use to make our elements display as if they were HTML tables
  • "Exotic": used less frequently, most people don't know about these, or they are simply deprecated (like box)
  • CSS-wide: values that are used on almost all CSS properties (not just display)

For the rest of this article, we will focus our attention on float property, and they way it works with the "block group" of values for the display property.

CSS float property vs CSS display property

It's time to discuss one of the most important concepts in CSS: the float property.

You can set the float property to:

  • left
  • right
  • none
  • inherit
  • initial
  • unset

Setting the float to right or left, will force the display to take the value of block, with a few specific consequences:

  • the floated element will become displayed as as block, but will not take up the full width of the screen
  • the element that comes after the floated element will start at the top and will need to be cleared.
  • the floated element's parent will not be "aware" of the floated element's height

That's a lot of concepts, so let's look at an example:

See the Pen Behavior of float:left or float:right by Ajdin (@AjdinImsirovic) on CodePen.

Looking at the example above, we can draw a number of conclusions:

  • The parent div's background is gray. The fact that the green-colored floated div is taller than it's parent is the proof that the parent div is not "aware" of the floated div's height
  • The second group of divs, which is 99% identical to the first group, is awkwardly positioned to the right of the first floated div, under the first group of pink divs. This is an example of why those div's need to be cleared.
  • The second group of divs honors whitespace, while the first group doesn't. Can you guess what's the pink divs' display property? Of course, it's inline-block. We are using a hack to fix this in the second group of divs. The hack is simple, we are using empty HTML comments. The most important point of the above example is that the parent div is not aware of the height of it's child, and that we need to clear floats.

Making the parent div aware of the height of the child div that has the float property

If a floated element's parent is part of the normal document flow (i.e. non-floated), the parent will "collapse", meaning it will not "wrap around" the child element.

Consider the following example:

See the Pen Non-floated parent div wrapping a floated child div by Ajdin (@AjdinImsirovic) on CodePen.

In the above example, we can see that the parent div does not wrap it's child div. In other words, the parent div is not "aware" of the child div's height.

The reason why this happens: The parent div does not contain a floated element. By the very definition, the floated element is not part of the normal document flow. It "floats" "above" it.

There are multiple ways to solve this:

  1. Float the parent div too
  2. Give the height to the parent div
  3. Add another element and give it the clear: both
  4. Add overflow: hidden to parent.
  5. Add .clearfix class on the parent div
  6. Imitate a regular div, using a combination of display: inline-block and width: 100%
  7. Give the parent div display: table
  8. Give the parent div pseudo-class of :after and add clearfix CSS there

Before we continue, let's briefly look at what clear: both, clear: left, and clear: right does.

The role of the clear property, according to W3C, can be found at this link. In a nutshell, when we give a clear property to a div, that div will place itself on the next fully clear width of the screen.

Here is an example of using clear: left.

See the Pen What does CSS clear property do? by Ajdin (@AjdinImsirovic) on CodePen.

The above example has 2 sections: the first one without clear CSS property on box 3, and the second one, with the clear property applied.

In the first example, we have the following setup:

  • The hollow boxes with gray borders (boxes 1 and 3) are not floated, i.e. they are a part of normal document flow.
  • The green box (box 2) is floated, and thus it is not a part of the normal document flow.
  • The red parent container is the parent div. It is aware of the height of only the hollow boxes (since they are part of the normal document flow).
  • The contents of box 3 is pushed to the right because of the space taken up by box 2.
  • Since the float is not cleared, even the next HTML element (the h2) is scooted over to the right, instead of taking up it's own place starting from the left edge of the screen.
  • If the green box was taller, it would usurp the space of even more HTML elements, and all of them would be pushed to it's right edge.

In the second example, we are using clear: left on box 3, which results in the following:

  • Box 3 is clearing the float of box 2, meaning, it is taking up the first possible position where it is able to stretch the full screen width of it's parent div.
  • Visually, we cannot know if box 2 is floated or not, because it behaves as if it is part of the normal document flow.
