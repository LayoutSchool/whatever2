CSS positioning is one of the basic skills that a frontend web developer needs to know.

The first thing to know is to have a general idea of how CSS positioning works. This includes knowing about the normal document flow in a web page: Left-to-right, top-to-bottom (of course, if your web page is in an RTL language, the flow will be right-to-left).

Most layouts are made by manipulating this normal document flow. To that end, we have multiple values for the position property in CSS. We'll look at all of them in detail in this article.

POSITION: STATIC

HTML elements start their life on a web page with the value of static on their position property. Static is just the normal positioning behavior of HTML elements.

See the Pen CSS positioning, static by Ajdin (@AjdinImsirovic) on CodePen.

Using static positioning, you cannot offset elements with top, right, bottom, and left properties. In order to be able to use these offset properties, you need a different kind of positioning.

POSITION: RELATIVE

Using the value of relative on the CSS position property, we now have at our displosal the four offset properties.

In the example that follows, we'll use the same code like we did in the previous example, with a slight twist: we'll change the positioning to relative, and we'll offset the wrapper by 100 pixels from the left side.

Note that we're also adding the offset property of left to box-2 and box-3.

See the Pen CSS positioning, relative by Ajdin (@AjdinImsirovic) on CodePen.

POSITION: ABSOLUTE

When you position an element absolutely, you take it out of the normal document flow.

For example, we have set the wrapping .box to position: static, and then .box-1 to position: absolute.

That way, we've taken it out of the normal document flow, so that the other divs are painted on the viewport as if the absolutely positioned element did not even exist.

This is what happens when you take an element out of the normal document flow.
