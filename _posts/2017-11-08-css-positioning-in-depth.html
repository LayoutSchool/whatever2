CSS positioning is one of the basic skills that a frontend web developer needs to know.

The first thing to know is to have a general idea of how CSS positioning works. This includes knowing about the normal document flow in a web page: Left-to-right, top-to-bottom (of course, if your web page is in an RTL language, the flow will be right-to-left).

CSS layouts are made by manipulating this normal document flow.

We have three major CSS properties at our disposal, to manipulate normal document flow in a web page. These properties are: float, position, and display.

In this article, we'll look at the position property in detail.

The available values for CSS position property

Position property in CSS can hold the following values:
  • static
  • relative
  • absolute
  • fixed
  • sticky
  • inherit
  • initial
  • unset

We will start this article by looking at what all of these position properties do and how to apply them in practice.

POSITION: STATIC

HTML elements start their life on a web page with the value of static on their position property. Static is just the normal positioning behavior of HTML elements.

POSITION: ABSOLUTE

When you position an element absolutely, you take it out of the normal document flow.

Absolutely positioning an element is as if you've tacked that element onto a web page.

For example, we have set the wrapping .box to position: static, and then .box-1 to position: absolute.

That way, we've taken it out of the normal document flow, so that the other divs are painted on the viewport as if the absolutely positioned element did not even exist.

However, since it does exist, and since it is the last div element on the page, it covers all the other divs that are using display:static, i.e. that are part of the normal document flow.

This is what happens when you take an element out of the normal document flow.

POSITION: FIXED

Giving an element's position property the value of fixed is similar to pasting that element on the screen itself.

So, while absolute positioned elements are positioned in terms of the page they are on, they still move with the page. When elements are fixed, they will remain in their place even as the page scrolls.

POSITION: STICKY

Position sticky is a combination of relative and fixed positioning principles. Basically, an element with position: sticky, behaves just like a relatively positioned element, until it reaches a certain part of the web page, at which time it behaves as if it's position was set as property:fixed.

POSITION: INHERIT

If you want an element to inherit it's parent's position property's value, you use position: inherit. What's interesting about the position property is that it does not cascade, so using position: inherit changes that default behavior.

POSITION: INITIAL

As a value, initial is used for any property in CSS. In other words, it is not only tied to the position property.

Basically, using the value of initial with any CSS property simply means that you want the browser to revert back to the default setting for that property.

Giving an element a value of position: initial is the same as if we gave it position: static, as position: static is the default value for all HTML elements.

POSITION: UNSET

Unset is sort of a hybrid value, in the sense that it is a combination of behaviors of the initial and inherit properties.

You can think of it as a sort of an if-statement, as the result of using it will be different based on the context it is used in.

    The way that it works is as follows:
  • If an element inherits the property from a parent element, using unset CSS value will revert the property to the inherited value.
  • If an element does not inherit the property from its parent element, using unset CSS value will revert the property to the initial value.

In the specific case of position property, we need to remember that the position property does not cascade, and thus, looking at the above 2 if statements, the second if statement applies. Simply put, using position: unset will be the same as if we used position: static.
